# Review sheet

## Key concepts
- **Subnet**: part of a larger IP network.
- **Routing table**: table used by routers and hosts to choose the next hop.
- **NAT**: translation from private addresses to a public address.
- **Switch**: forwards frames within the LAN.
- **Router**: connects different networks.
- **RIP**: simple routing protocol based on hop count.
- **BGP**: routing protocol between autonomous systems on the Internet.
- **Reliable data transfer**: ACKs, timeouts, retransmission, and windowing.
- **HTTP**: application protocol for request/response traffic.
- **TLS**: cryptographic protection on top of transmission.

## Recommended presentation order
1. Observe the path of a packet from host to host.
2. Identify where the subnet comes into play.
3. Distinguish switches from routers.
4. Add reliability and transport.
5. Finish with HTTP, REST, and TLS.

## Review questions with short answers
- **What is a subnet?** A logical partition of an IP network.
- **What is routing?** The choice of a path toward a destination.
- **Why is broadcast not suitable for the Internet?** Because it is limited to the local network.
- **Why is TLS important?** To protect confidentiality and integrity.
